Configuration
Learn how to configure the framework for different analysis scenarios. 
# Example: Custom configuration for sensitivity analysis
from analysis.core.config import V4Config, EngineConfig, AsyncConfig

# Create custom configuration
config = V4Config.from_yaml("config/v4_analysis_defaults.yml")

# Customize engine settings
config.engines.execution_mode = "asynchronous"
config.engines.max_workers = 8
config.async_config.max_concurrent_engines = 6

# Enable performance monitoring
config.performance_monitoring.enable_profiling = True
config.performance_monitoring.profile_memory = True

# Save custom configuration
config.to_yaml("config/custom_analysis.yml")
Best Practices
Follow these best practices for optimal performance and maintainability.
Performance Tips
  • Use async execution for large datasets (>10000 draws)
  • Enable caching when running multiple similar analyses
  • Monitor memory usage with built-in performance tracking
  • Validate inputs before running expensive analyses
Code Organization
  • Use descriptive variable names for clarity
  • Handle exceptions gracefully with proper error messages
  • Document custom engines with comprehensive docstrings
  • Follow consistent naming conventions across your code
Data Management
  • Validate data integrity before analysis
  • Use provenance tracking for reproducible research
  • Export metadata with your results
  • Archive intermediate results for debugging
